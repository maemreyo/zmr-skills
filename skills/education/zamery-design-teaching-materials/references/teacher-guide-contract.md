# Teacher Guide Contract

The guide is a lesson-control surface, not a project report. Use three or four pages with four represented roles:

1. `command_center`: lesson title, grade, duration, objectives, success evidence, materials, preparation, and misconceptions.
2. `teach`: minute-by-minute flow, teacher moves, expected student evidence, transitions, and board plan.
3. `respond`: checks for understanding, support, challenge, differentiation, and recovery moves.
4. `answers`: teacher-only answers with a narrow item column, answer, rationale, and source version.

Keep sections scannable during live teaching. Exclude the user's prompt, route plan, raw brief fields, manifests, QA prose, and implementation notes. Use asymmetric tables only for repeated comparable records; never allocate equal width to stage, minutes, and teacher move.
